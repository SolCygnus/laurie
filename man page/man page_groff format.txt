.TH MAIN 1 "January 2025" "Version 1.0" "Custom Linux Hardening Script"
.SH NAME
main.py \- A Python-based Linux VM hardening and application installation manager.
.SH SYNOPSIS
.B python3 main.py
[OPTIONS]
.SH DESCRIPTION
The \fImain.py\fP script automates the hardening and configuration of new Linux virtual machines. It manages:
.IP \(bu 2
Hardening (firewall, user account security, system updates).
.IP \(bu 2
Application installation (e.g., Firejail, Fail2Ban, Lynis).
.IP \(bu 2
Customized system settings (e.g., UFW, AppArmor).
.PP
After running, users can refer to this man page for a list of installed applications and troubleshooting tips.
.SH OPTIONS
.TP
\fB\-h\fP, \fB\-\-help\fP
Show help and usage information.
.TP
\fB\-\-install\fP
Install all necessary applications and apply hardening.
.TP
\fB\-\-check\fP
Display the current hardening status and logs.
.SH INSTALLED APPLICATIONS
The script installs and configures the following:
.IP \(bu 2
UFW (firewall with strict rules).
.IP \(bu 2
Fail2Ban (intrusion prevention).
.IP \(bu 2
Firejail (application sandboxing).
.IP \(bu 2
Lynis (security auditing).
.PP
.SH LOGS
Check the following logs for troubleshooting:
.IP \(bu 2
UFW logs: \fI/var/log/ufw.log\fP
.IP \(bu 2
Audit logs: \fI/var/log/audit/audit.log\fP
.IP \(bu 2
Fail2Ban logs: \fI/var/log/fail2ban.log\fP
.SH COMMON COMMANDS
.TP
Check UFW status:
.B sudo ufw status verbose
.TP
Restart Fail2Ban:
.B sudo systemctl restart fail2ban
.TP
Run Lynis audit:
.B sudo lynis audit system
.SH SEE ALSO
.B ufw(8), fail2ban(8), lynis(8)
